# 中国结云展览项目需求文档（PRD）

## 一、项目概述

### 1.1 项目背景
中国结是中国传统文化的重要符号之一，承载着深厚的文化内涵和美好的祝愿。为了更好地传承和弘扬中国结艺术，让更多人了解和欣赏中国结之美，我们计划开发一个"中国结云展览"线上展览平台。

### 1.2 项目目标
- **文化传播**：通过数字化手段展示中国结艺术，让传统文化触达更广泛的受众
- **教育普及**：系统化介绍不同类型中国结的历史、技法和文化寓意
- **互动体验**：提供沉浸式的观展体验，让用户随时随地欣赏中国结艺术

### 1.3 项目定位
一个面向大众的中国结艺术在线展览平台，基于 Web 技术（H5），支持桌面和移动浏览器访问。

### 1.4 目标用户
- **文化爱好者**：对中国传统文化感兴趣的人群
- **学生群体**：学习传统文化、手工艺的学生
- **手工爱好者**：想要学习中国结编织的人群
- **游客**：寻找文化体验的游客

## 二、功能需求

### 2.1 页面架构

项目采用三页式架构：
```
首页（Home） → 展览页（Gallery） → 详情页（Detail）
```

### 2.2 首页（Home）

#### 2.2.1 功能描述
首页作为项目的门户页面，营造中国传统文化氛围，引导用户进入展览。

#### 2.2.2 页面元素
- **顶部导航栏**
  - Logo图标
  - 项目标题

- **主视觉区**
  - 背景：中国结装饰元素（飘带、祥云等）
  - 主标题："心有千千结 云上共此时"
  - 副标题/简介：简要介绍中国结文化和本次展览

- **交互按钮**
  - "进入展览"按钮：跳转到展览页

- **装饰元素**
  - 中国风装饰图案
  - 动态飘带效果（可选）

#### 2.2.3 交互逻辑
- 点击"进入展览"按钮 → 跳转到展览页（Gallery）
- 页面加载时有淡入动画效果

---

### 2.3 展览页（Gallery）

#### 2.3.1 功能描述
展览页是核心功能页面，展示所有中国结作品，支持分类浏览和筛选。

#### 2.3.2 页面布局
采用左右分栏布局：
- **左侧：分类导航区**（约20%宽度）
- **右侧：作品展示区**（约80%宽度）

#### 2.3.3 分类导航区
- **分类列表**（垂直排列）
  - 艺术结
  - 吉祥结
  - 装饰结
  - 实用结

- **交互效果**
  - 当前选中分类高亮显示
  - 点击分类切换展示内容
  - 默认显示"艺术结"分类

#### 2.3.4 作品展示区
- **卡片式网格布局**
  - 每行显示3-4个作品卡片（响应式）
  - 卡片间距均匀，视觉整洁

- **作品卡片元素**
  - 中国结图片（正方形或固定比例）
  - 作品名称
  - 简短描述（可选）
  - 悬停/点击效果

- **交互逻辑**
  - 点击作品卡片 → 打开详情页弹窗
  - 卡片悬停时有缩放或阴影效果
  - 支持图片懒加载优化性能

#### 2.3.5 扩展功能（可选）
- 搜索框：按名称搜索作品
- 筛选器：按难度、材质等筛选
- 分页或无限滚动加载

---

### 2.4 详情页（Detail）

#### 2.4.1 功能描述
详情页以弹窗形式展示单个中国结作品的详细信息，提供沉浸式查看体验。

#### 2.4.2 页面布局
模态弹窗设计，分为左右两栏：
- **左侧：展示区**（约60%宽度）
- **右侧：信息区**（约40%宽度）
- **背景：模糊蒙层**（暗化Gallery页面）

#### 2.4.3 展示区（左侧）
- **3D模型交互展示**（核心功能）
  - 中国结3D模型实时渲染
  - **支持任意角度旋转查看**：用户可通过鼠标拖拽或触摸滑动，360度全方位观看中国结
  - **支持缩放查看**：用户可通过滚轮或双指缩放，近距离观察中国结细节
  - **自动旋转模式**（可选）：3D模型缓慢自动旋转展示
  - 深色或中性背景，突出作品，营造展览氛围
  - 高性能渲染，保证流畅交互体验

- **交互控件**
  - 旋转控制：鼠标拖拽/触摸滑动旋转
  - 缩放控制：滚轮/双指捏合缩放
  - 重置视角按钮：一键恢复默认视角
  - 自动旋转开关（可选）
  - 上一个/下一个作品切换按钮（可选）

#### 2.4.4 信息区（右侧）
- **标题区**
  - 作品名称（大号字体）
  - 分类标签
  - 关闭按钮（右上角）

- **详细信息**（可滚动）
  - **简介**：作品的基本描述
  - **历史渊源**：中国结的历史背景
  - **编织技法**：制作工艺介绍
  - **文化寓意**：象征意义和祝福含义
  - **材料信息**：所需材料列表
  - **难度等级**：展示难度星级

- **操作按钮**（可选）
  - "收藏"按钮
  - "分享"按钮
  - "查看教程"按钮

#### 2.4.5 交互逻辑
- 点击Gallery页作品卡片 → 弹出详情页弹窗
- 点击关闭按钮/蒙层 → 关闭弹窗返回Gallery
- 支持键盘ESC关闭
- 左右箭头切换上下作品（可选）
- 弹窗打开/关闭有动画效果

## 三、内容需求

### 3.1 中国结分类

#### 艺术结
以艺术欣赏为主，造型复杂精美，如：盘长结、团锦结等

#### 吉祥结
寓意吉祥美好，常用于节日庆典，如：如意结、吉祥结等

#### 装饰结
用于装饰搭配，实用美观，如：纽扣结、酢浆草结等

#### 实用结
具有实用功能的基础结，如：平结、双钱结等

### 3.2 作品信息要求

每个中国结作品至少包含：
- **3D模型文件**（必需）：支持GLB/GLTF格式，建议文件大小控制在2MB以内
- 高清图片（建议分辨率：800x800px以上，用于缩略图和备用展示）
- 名称
- 所属分类
- 详细介绍（300-500字）
- 历史渊源
- 文化寓意
- 编织技法说明

### 3.3 内容数量规划
- 艺术结：6-8个作品
- 吉祥结：6-8个作品
- 装饰结：6-8个作品
- 实用结：6-8个作品
- **总计**：24-32个作品

## 四、非功能需求

### 4.1 性能需求
- **页面加载时间**：首页3秒内完成加载
- **3D模型加载**：单个模型加载时间控制在3秒以内，显示加载进度
- **3D渲染性能**：保证60fps流畅渲染，旋转和缩放操作无延迟
- **图片加载**：支持懒加载，优化大图展示
- **流畅度**：页面切换和动画流畅，无卡顿

### 4.2 兼容性需求
- **浏览器支持**
  - Chrome 90+（推荐）
  - Firefox 88+
  - Safari 14+
  - Edge 90+
  - 移动端浏览器：iOS Safari、Chrome Mobile、微信内置浏览器

- **设备支持**
  - 桌面浏览器（1920x1080、1366x768 等主流分辨率）
  - 平板设备（iPad 等）
  - 移动端（375px - 768px 宽度，响应式适配）

- **技术要求**
  - 必须支持 WebGL（用于 3D 渲染）
  - 推荐使用支持硬件加速的现代浏览器

### 4.3 用户体验需求
- **视觉风格**：中国传统文化风格，配色典雅
- **色彩方案**
  - 主色：中国红（#C8102E）、金色（#D4AF37）
  - 辅助色：米白（#F5F5DC）、墨黑（#1C1C1C）
  - 背景色：浅米色或浅灰（#FAF9F6）

- **字体选择**
  - 标题：中文宋体或书法字体
  - 正文：系统默认中文字体，易读性好

- **动画效果**
  - 页面切换：淡入淡出、滑动效果
  - 元素加载：渐显动画
  - 交互反馈：悬停、点击有明显反馈

### 4.4 可访问性需求
- 图片提供alt文本
- 色彩对比度符合WCAG标准
- 支持触摸和鼠标操作

## 七、风险与挑战

### 7.1 技术风险
- **3D渲染性能**：3D模型在低端设备上可能存在性能问题，需要优化
- **3D模型文件大小**：模型过大会影响加载速度，需要压缩优化
- **浏览器兼容性**：旧版浏览器不支持 WebGL 2.0，需要降级方案
- **移动端性能**：移动设备 GPU 性能有限，需要针对性优化
- **触控交互**：移动端触摸手势识别的准确性和流畅度

### 7.2 资源风险
- **3D模型制作**：需要专业3D建模人员制作24-32个高质量中国结模型
- **模型质量控制**：模型精度、贴图质量、文件大小需要平衡
- **内容编写**：专业文案需要一定的文化知识储备

### 7.3 应对方案
- **3D模型获取**：
  - 方案1：委托专业建模师制作（推荐，质量可控）
  - 方案2：使用摄影测量技术生成3D模型
  - 方案3：购买/下载现成的中国结3D模型并优化
- **性能优化**：
  - 模型面数控制在5000面以内
  - 使用压缩纹理（Draco、KTX2）
  - 实现LOD（细节层次）技术，根据距离切换模型精度
  - 添加加载进度提示，提升用户体验
  - 使用 Web Worker 在后台线程处理模型加载
- **兼容性方案**：
  - 检测浏览器 WebGL 支持情况
  - 不支持 WebGL 时降级为高清图片展示
  - 提供浏览器升级提示
- **分阶段上线**：
  - Phase 1：桌面浏览器优先（开发和测试）
  - Phase 2：移动端浏览器适配和优化
  - Phase 3：性能调优和用户体验提升

## 十一、3D功能技术要点补充

### 11.1 3D模型规格要求
- **格式**：GLB（推荐）或GLTF
- **面数**：每个模型3000-5000面
- **贴图分辨率**：1024x1024或2048x2048
- **文件大小**：单个模型控制在1-2MB
- **坐标系**：采用Y-up右手坐标系
- **缩放比例**：统一缩放，保证不同模型大小一致

### 11.2 3D交互功能列表
| 功能 | 操作方式（桌面端） | 操作方式（移动端） | 优先级 |
|------|-------------------|-------------------|--------|
| 旋转模型 | 鼠标左键拖拽 | 单指滑动 | P0（必须） |
| 缩放模型 | 鼠标滚轮 | 双指捏合 | P0（必须） |
| 平移模型 | 鼠标右键拖拽 | 双指滑动 | P1（建议） |
| 重置视角 | 点击重置按钮 | 点击重置按钮 | P0（必须） |
| 自动旋转 | 点击开关按钮 | 点击开关按钮 | P1（建议） |
| 切换光照 | 点击开关按钮 | 点击开关按钮 | P2（可选） |

### 11.3 3D场景配置
- **相机类型**：透视相机（PerspectiveCamera）
- **初始视角**：正面45度斜视角
- **相机距离**：根据模型大小自适应调整
- **相机控制**：OrbitControls（轨道控制器）
  - 启用阻尼（damping）实现平滑交互
  - 限制旋转角度（防止倒置）
  - 限制缩放范围（minDistance / maxDistance）
- **光照方案**：
  - 环境光（AmbientLight）：柔和整体照明，强度 0.4-0.6
  - 方向光（DirectionalLight）：主光源，模拟自然光，强度 0.8-1.2
  - 点光源（PointLight，可选）：增强细节表现
- **背景**：深灰色渐变背景（#2a2a2a → #1a1a1a），突出模型
- **抗锯齿**：启用 antialias
- **阴影**：可选启用（影响性能）

### 11.4 加载优化策略
- 显示加载进度条（0-100%）
- 模型加载失败时显示默认图片
- 使用Draco压缩减小文件体积
- 实现模型预加载（浏览下一个作品时提前加载）
